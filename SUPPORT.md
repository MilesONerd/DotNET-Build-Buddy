# Support for DotNET Build Buddy

Thank you for using **DotNET Build Buddy**! Here’s how you can get help and interact with the community.

## Where to Ask for Help

For documentation:
- Use the **[docs/ directory](https://github.com/MilesONerd/DotNET-Build-Buddy/tree/master/docs)** in this repository.

For general questions, discussions, or suggestions:  
- Use the **[Discussions](https://github.com/MilesONerd/DotNET-Build-Buddy/discussions)** section of this repository.

For bug reports, feature requests, or issues with the extension:  
- Open an **[Issue](https://github.com/MilesONerd/DotNET-Build-Buddy/issues)** in this repository. 

For security vulnerabilities or sensitive issues:  
- Email us at **security@milesonerd.8shield.net** 

> Please note: We do **not provide support for forks** of this project. All support is limited to the official repository.

## Frequently Asked Questions

Before opening a discussion or issue, we recommend checking our **[documentation](https://github.com/MilesONerd/DotNET-Build-Buddy#readme)** and previous discussions — your question may already be answered!

## Community Guidelines

- Be respectful and clear in your questions or reports.
- Provide relevant details such as VS Code version, DotNET Build Buddy version, and any error messages.  
- Avoid asking for support on personal forks, as we cannot guarantee compatibility.
- Follow de [Contributor Covenant Code of Conduct](/CODE_OF_CONDUCT.md)

We love engaging with the community and helping everyone get the most out of **DotNET Build Buddy**!
